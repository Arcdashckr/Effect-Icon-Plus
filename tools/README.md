# 🛠️ Effect-Icon-Plus Developer ToolKit

This folder is a fully-fledged, Python-powered automation studio designed to download mods/collections from Modrinth, check and fetch mod metadata, add new mod compatibility to the **Effect-Icon-Plus** resource pack, manage color palettes, and automatically update repository displays and documentation.

All processes are interconnected via Python scripts leveraging the ImageMagick library for optimal workflow efficiency.

---

## 🚀 Quick Start

To manage the entire toolkit from a single dashboard, simply double-click and run **`start.bat`** inside the folder. You will be greeted by the following integrated menu:

- [1] Download Mod or Collection
- [2] Extract and Scale Effect Icons
- [3] Merge Frames
- [4] Create Display Grid
- [5] Generate Repository Files
- [6] Run Full Pipeline
- [7] Clean Temporary Files
- [8] Exit

---

## 📦 Step-by-Step Workflow & File Responsibilities

### 0️⃣ Step 0: Downloader & Metadata Manager (collection_downloader.py & metadata_manager.py)
* **collection_downloader.py**:
  * Downloads entire Modrinth collections or a single mod via slug/URL.
  * Only retains mods that actually contain status effect icons (e.g. checks for a `mob_effect` texture folder inside the JAR before keeping it).
  * Triggers metadata updates dynamically.
* **metadata_manager.py**:
  * Extracts namespaces from JARs and manages the `mod_metadata.json` color/author/icon cache.
  * Offers an interactive console matching interface: if a mod namespace doesn't directly match a Modrinth slug, it performs a search, displays potential matches, and asks the user to choose the correct one, input a slug manually, or skip.

### 1️⃣ Step 1: Icon Extraction (extract_and_scale.py)
* **Purpose:** Automatically scans and detects effect icons inside `.jar` or `.zip` mod files in `tools/mods/`.
* **How it works:** Reads the mod archive, captures all 16x16 pixel-art icons inside the `assets/<mod_id>/textures/mob_effect/` directory, scales them crisply (Nearest Neighbor) to **36x36**, and saves them under `tools/extracted/<mod_id>/`.

### 2️⃣ Step 2: Color Analysis & Framing (merge_frames.py)
* **Purpose:** Extracts the color palette from raw icons, populates `colors.json`, and merges the 9-stage colored frame slices (`frames/`) onto the icon.
* **Advanced Features:**
  * **Speed Filter:** Prompted on run. If an output icon already exists in the repository and the JSON remains unchanged, it skips processing—saving immense amounts of time.
  * **Contrast Booster:** Prevents ImageMagick's auto-generated colors from turning out dull or greyish by mathematically expanding the gradient range to make the borders pop.
  * **Dual-Block Management (colors.json):**
    * "auto": The untouched reference block generated by ImageMagick, mapped directly to the 9 border frames.
    * "manual": Reserved for your custom hex entries. Additionally, an `_all` key appended to the effect name stores all unique raw colors found in the icon for easy reference (does not affect rendering).

### 3️⃣ Step 3: Display Automation (create_display.py)
* **Purpose:** Arranges the completed 54x54 framed icons into a 10-column layout preview sheet.
* **Options:**
  1. *Vanilla Only:* Scrapes `assets/minecraft` and outputs `effect_display_vanilla.png`.
  2. *Specific Mod:* Generates a layout exclusively for a single manually typed mod folder.
  3. *All Mods Separately:* Generates distinct sheets for each independent mod folder.
  4. *Merge Everything:* Unifies Vanilla and mods into one giant `effect_display_mod.png` sheet.
* **Advanced Features:**
  * **Lossless Dynamic Scaling:** Dynamic vertical stretch based on icon count, crisp-scaling pixels by 300% (`_upscaled.png`) for a flawless GitHub showcase.

### 4️⃣ Step 4: Repository Documentation (generate_repo_files.py)
* **Purpose:** Automatically regenerates repository documentation and metrics.
* **Outputs:**
  * **Live README Integration (`stats.json`):** Tracks total mod and icon counters for root README badges.
  * **Automated `compatibility.md`:** Resolves cached metadata (`mod_metadata.json`) to compile an alphabetized Markdown compatibility table, utilizing `page_url` to link back to each mod's page.

### 5️⃣ Step 5: Cleanup (clean_temp.py)
* **Purpose:** Cleans up work-in-progress files inside `tools/mods/` and `tools/extracted/` directories to save disk space, while preserving the `.gitignore` files inside them.

## 🛠️ Requirements

The following dependencies must be installed on your system and appended to your Environment Variables (PATH):
1. **Python 3.x**
2. **ImageMagick** (The global `magick` command must be executable via terminal.)

## 📁 Repository Structure Reference

- tools/
  - extracted/           # 36x36 raw mod icons from Step 1
  - frames/              # White border frame slices from 1.png to 9.png
  - mods/                # Downloaded jar/zip mods
  - start.bat            # Main control panel script
  - colors.json          # Color database (auto/manual)
  - mod_metadata.json    # Mod metadata cache
  - collection_downloader.py # Mod downloader
  - metadata_manager.py  # Metadata manager
  - extract_and_scale.py # Mod extractor script
  - merge_frames.py      # Smart color-mapping and frame compiler script
  - create_display.py    # Display updater script
  - generate_repo_files.py # Stats and compatibility table updater script
  - clean_temp.py        # Temporary files cleaner
